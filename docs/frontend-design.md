# PortTask Frontend Design Specification

## Overview

PortTask is a next-generation maritime operations platform consisting of three interconnected portals:

| Portal | Domain | Audience |
|--------|--------|----------|
| Main Platform | `key.porttask.com` | Internal operators |
| Operations Portal | `portal.porttask.com` | Customers, crew, agents |
| Vendor Portal | `vendor.porttask.com` | Service providers |

---

## Design System

### Brand Colors

```
Primary:
  Navy Blue:      #0F2B4C (headers, primary actions)
  Ocean Blue:     #1E5FA8 (links, interactive elements)
  Sky Blue:       #4A9FD4 (highlights, hover states)

Secondary:
  Slate:          #64748B (secondary text)
  Light Gray:     #F1F5F9 (backgrounds)
  White:          #FFFFFF (cards, content areas)

Status:
  Success:        #10B981 (completed, positive)
  Warning:        #F59E0B (pending, attention)
  Error:          #EF4444 (failed, urgent)
  Info:           #3B82F6 (informational)

Maritime Accent:
  Deep Sea:       #0C4A6E (dark mode, emphasis)
  Foam:           #E0F2FE (light backgrounds, tags)
```

### Typography

```
Font Family:
  Primary:        Inter (UI elements)
  Monospace:      JetBrains Mono (data, codes)

Font Sizes:
  xs:             12px (labels, captions)
  sm:             14px (body text, inputs)
  base:           16px (default)
  lg:             18px (subheadings)
  xl:             20px (section titles)
  2xl:            24px (page titles)
  3xl:            30px (dashboard KPIs)

Font Weights:
  Regular:        400
  Medium:         500
  Semibold:       600
  Bold:           700
```

### Spacing System

```
Base unit: 4px

Space scale:
  1:    4px
  2:    8px
  3:    12px
  4:    16px
  5:    20px
  6:    24px
  8:    32px
  10:   40px
  12:   48px
  16:   64px
```

### Border Radius

```
sm:     4px   (buttons, inputs)
md:     8px   (cards, modals)
lg:     12px  (large cards, panels)
xl:     16px  (featured sections)
full:   9999px (avatars, badges)
```

### Shadows

```
sm:     0 1px 2px rgba(0,0,0,0.05)
md:     0 4px 6px rgba(0,0,0,0.07)
lg:     0 10px 15px rgba(0,0,0,0.1)
xl:     0 20px 25px rgba(0,0,0,0.15)
```

### Component Library

#### Buttons

| Type | Use Case | Style |
|------|----------|-------|
| Primary | Main actions | Navy background, white text |
| Secondary | Alternative actions | White background, navy border |
| Ghost | Tertiary actions | Transparent, navy text |
| Danger | Destructive actions | Red background, white text |
| Icon | Icon-only actions | Circular, ghost style |

#### Cards

```
Standard Card:
  - Background: white
  - Border: 1px solid #E2E8F0
  - Border-radius: 8px
  - Padding: 24px
  - Shadow: sm

Interactive Card:
  - Hover: shadow-md, translateY(-2px)
  - Cursor: pointer
  - Transition: 150ms ease

Stat Card:
  - Large number (3xl, bold)
  - Label below (sm, slate)
  - Optional icon/trend indicator
```

#### Tables

```
Header:
  - Background: #F8FAFC
  - Font-weight: 600
  - Text-transform: uppercase
  - Font-size: xs
  - Letter-spacing: 0.05em

Rows:
  - Border-bottom: 1px solid #E2E8F0
  - Hover: background #F8FAFC
  - Padding: 16px vertical

Actions Column:
  - Right-aligned
  - Icon buttons or dropdown menu
```

#### Forms

```
Input Fields:
  - Height: 40px
  - Border: 1px solid #CBD5E1
  - Border-radius: 6px
  - Padding: 0 12px
  - Focus: border-color Ocean Blue, ring

Labels:
  - Font-size: sm
  - Font-weight: 500
  - Margin-bottom: 6px

Error State:
  - Border-color: Error red
  - Helper text below in red
```

#### Navigation

```
Sidebar (Main Platform):
  - Width: 260px (expanded), 72px (collapsed)
  - Background: Navy Blue
  - Text: white/70% opacity
  - Active item: white, left border accent

Top Bar:
  - Height: 64px
  - Background: white
  - Border-bottom: 1px solid #E2E8F0
  - Contains: search, notifications, profile

Tabs:
  - Underline style
  - Active: Ocean Blue underline, semibold
  - Inactive: Slate text
```

## Responsive Design

### Breakpoints

```
Mobile:     < 640px
Tablet:     640px - 1024px
Desktop:    1024px - 1440px
Large:      > 1440px
```

### Mobile Adaptations

**Main Platform:**
- Sidebar collapses to hamburger menu
- Tables become cards
- Multi-column layouts stack

**Portals:**
- Tab navigation becomes bottom nav or hamburger
- Cards stack vertically
- Actions move to floating buttons

### Touch Targets

- Minimum touch target: 44px Ã— 44px
- Spacing between touch targets: 8px minimum

---

## Accessibility

### Standards
- WCAG 2.1 Level AA compliance
- Keyboard navigation support
- Screen reader compatibility

### Requirements
- Color contrast ratio: 4.5:1 minimum
- Focus indicators on all interactive elements
- Alt text for all images
- ARIA labels for complex components

---

## Performance

### Targets
- First Contentful Paint: < 1.5s
- Time to Interactive: < 3s
- Lighthouse score: > 90

### Strategies
- Code splitting by route
- Lazy loading for images
- Virtualized lists for large datasets
- Service worker for offline capability

---

## Error States

### Empty States
- Friendly illustration
- Clear message
- Call to action

### Error Messages
- Descriptive error text
- Suggested resolution
- Support contact option

### Loading States
- Skeleton screens for content
- Spinners for actions
- Progress indicators for long operations

---

## Notifications

### Types
| Type | Use | Duration |
|------|-----|----------|
| Success | Confirmations | 3 seconds, auto-dismiss |
| Error | Failures | Persist until dismissed |
| Warning | Cautions | 5 seconds |
| Info | Information | 4 seconds |

### Channels
- In-app notifications (bell icon)
- Push notifications (mobile)
- Email notifications (configurable)
